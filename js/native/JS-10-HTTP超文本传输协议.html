# HTTP

超文本传输协议（HyperText Transfer Protocol，HTTP） 是一种用于分布式、协作式和超媒体信息系统的应用层协议

# 1. HTTP 的特点

  • '请求-响应'模式

客户端先发出请求，然后服务器端响应该请求并返回

  • 无状态

无状态是指协议对于事务处理没有记忆能力 而为了解决HTTP的无状态特性，出现了 Cookie 和 Session 技术

  • 无连接

无连接的含义是限制每次连接只响应一个请求，然后就断开连接

# 2. 报文

HTTP中的报文(message)是网络中交换与传输的数据单元，即站点一次性要发送的数据块 报文包含请求/响应行，请求/响应头，请求/响应体

  • 请求报文
请求方法 请求路径 HTTP版本
请求头
请求体
  • 响应报文
HTTP版本 响应状态码 状态码的原因短语
响应头
响应体

# 2.1 请求方法

请求方法 说明
GET 获取请求, 无请求体, 是幂等的
POST 添加请求, 可以有请求体, 是幂等的
HEAD 获取请求，但服务器将不传回资源的文本部分
PUT 修改请求, 可以有请求体, 是幂等的
DELETE 删除请求, 可以有请求体, 是幂等的
TRACE 请求测试或诊断
OPTION 请求方法查询
CONNECT 预留给代理服务器

# 2.2 请求路径

HTTP的URL(Uniform Resource Locator)是一种特殊类型的URI(Uniform Resource Identifier)

protocol://host[':'port][abs_path]['?'query][fragment]
protocol://host[':'port][abs_path][':'params][fragment]

# 2.3 请求/响应头

包含对请求和响应进行限制和说明若干字段

字段 说明 示例
Accept 可接收的响应内容类型 Accept:text/plain (文本类型)
Accept-Charset 可接收的字符集 Accept-Charset: utf-8
Accept-Encoding 可接收的响应内容的编码方式 Accept-Encoding: gzip, deflate
Accept-Language 可接收的响应内容语言列表 Accept-Language: en-US
Accept-Datetime 可接收的按照时间来表示的响应内容版本 Accept-Datetime: Sat, 26 Dec 2015 17:30:00 GMT
Authorization HTTP协议中需要认证资源的认证信息 Authorization: Basic OSdjJGRpbjpvcGVuIANlc2SdDE==
Cache-Control 请求/响应中,是否使用缓存机制 Cache-Control: no-cache
Connection 客户端想要优先使用的连接类型 Connection: keep-alive / Upgrade
Content-Length 以8进制表示的请求体的长度 Content-Length: 348
Content-Type 请求体的MIME类型 Content-Type: application/x-www-form-urlencoded / application/json
Date 发送该消息的日期和时间 Date: Dec, 26 Dec 2015 17:30:00 GMT
E-tag 表示缓存唯一ID E-tag:
Expect 表示客户端要求服务器做出特定的行为 Expect: 100-continue
Expires 表示缓存过期时间 Expires: 3600
From 发起此请求的用户的邮件地址 From: user@itbilu.com
Host 服务器域名和端口号,默认端口可省略 Host: www.itbilu.com:80 or www.itbilu.com
If-Match 主要用于PUT,实体匹配才可以操作 If-Match: "9jd00cdj34pss9ejqiw39d82f20d0ikd"
If-Modified-Since 资源未被修改的情况下返回304未修改 If-Modified-Since: Dec, 26 Dec 2015 17:30:00 GMT
If-None-Match 资源与缓存ID不匹配时剋操作 If-None-Match：200
Last-Modify 缓存最后的修改时间 Last-Modify：Sat, 26 Dec 2015 17:30:00 GMT
User-Agent 浏览器的身份标识字符串 User-Agent: Mozilla/...
Upgrade 要求服务器升级到一个高版本协议 Upgrade: HTTP/2.0, SHTTP/1.3, IRC/6.9, RTA/x11
Via 告诉服务器,这个请求是由哪个代理发出的 Via: 1.0 fred, 1.1 itbilu.com.com (Apache/1.1)
Referer 表示跳转到当前页面的之前的页面 Referer: http://news.baidu.com
Origin 发起一个针对跨域资源共享的请求 Origin: http://www.baidu.com

# 2.4 请求/响应体

包含请求参数和响应数据/内容

# 2.5 响应状态码

类别 原因短语
1XX Imformation (信息性状态码) 接收的请求正在处理
2XX Success (成功状态码) 请求正常处理完毕
3XX Redirection (重定向状态码) 需要进行附加操作以完成请求
4XX Client Error (客户端错误状态码) 服务器无法处理请求
5XX Server Error (服务器错误状态码) 服务器处理请求出错

# 3. HTTP 中的缓存

# 3.1 强制缓存

// 响应头
Cache-control: public, max-age=3600
// 请求头 
Expires: 3600

# 3.2 协商缓存

// 响应头
E-tag: xxxx
// 请求头
If-None-Match : 304, 200

// 响应头
Last-Modify: time 
// 请求头
If-Modify-Since: time

Cache-Control

用在请求头部: no-cache: 客户端告知本次请求不接收缓存数据, 直接请求源服务器 no-store: 客户端告知本次请求和响应内容不要被缓存 max-age=[秒]: 客户端告知本次请求有条件的接收缓存数据: 缓存时间不超过指定值, 否则请求源服务器 max-stale=([秒]): 有值的情况: 同 max-age; 无值的情况: 客户端告知本次请求接收所有缓存数据 min-fresh=[秒]: 客户端告知本次请求会筛选缓存数据: 只接收在指定时间内不会过期的, 其余不要返回 only-if-cached: 客户端告知本次请求只接收缓存数据, 否则返回 504 must-revalidate: 客户端告知本次请求有条件的接收缓存数据: 必须再次验证缓存有效, 否则返回 504 no-transfer: 客户端告知本次请求的缓存数据的实体的类型不能被转换

用在响应头部: no-cache: 服务端告知本次响应内容缓存前必须再验证有效性 no-store: 服务端告知本次请求和响应内容不要被缓存 max-age=[秒]: 服务端告知本次响应内容缓存后不用再验证: 只要缓存时间不超过指定值 s-maxage=[秒]: 同 max-age, 只针对公共缓存服务器 public: 服务端告知本次响应内容缓存后可被其他用户使用 private: 服务端告知本次响应内容缓存后不能被其他用户使用 must-revalidate: 同 no-cache proxy-revalidate: 同 no-cache, 针对所有缓存服务器 no-transfer: 服务端告知本次响应的内容缓存后的实体的类型不能被转换

# 4. HTTPS

# 4.1 HTTP 的缺点

  1. 明文通信 -> 窃听
  2. 无身份验证 -> 伪装
  3. 无信息完整性验证 -> 篡改

# 4.2 HTTPS = ?

HTTPS = HTTP + SSL3.0 + TLS1.0 即 HTTPS = HTTP + 加密 + 认证 SSL: Secure Socket Layer 协议 TLS: Transport Layer Security 协议

# 4.3 共享密钥和公开密钥

  • 共享密钥

加密和解密使用同一把密钥 优点: 速度较快 缺点: 无法保证密钥在传输过程中的安全性

  • 公开密钥

加密和解密使用不同的密钥, 私钥服务器保存, 公钥配发给客户端 优点: 不需叫密钥来回传送 缺点: 速度较慢; 无法验证公钥配发的安全性

通信时, 先用公开密钥加密共享密钥进行安全传递, 之后采用共享密钥通信

# 4.4 认证

  • 公钥证书

第三方数字证书认证机构 (CA) 颁发 认证流程: => 浏览器预先将机构自己的公钥内置到浏览器中 => 服务器将要配发的自己的公钥交给机构 => 机构用自己的私钥给服务器交给它的公钥进行加密数字签名 => 机构将加密后的服务器公钥和公钥证书一起发送给客户端 => 客户端用浏览器内置的机构公钥对数字签名进行验证 => 验证无误后客户端即可使用服务器公钥

  • EV SSL 证书

Extended Validation SSL Certificate 验证服务器拥有组织的真实性

  • 客户端证书

用在一些特殊的需要验证客户端真实性的业务上

# 4.5 HTTP or HTTPS ?

考虑到成本和性能的问题, 只在必要的通信上使用 HTTPS, 其他依然使用 HTTP
