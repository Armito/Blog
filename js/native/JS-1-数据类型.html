# 数据类型

# 一. 简单/基本/原始数据类型

String, Number, Boolean, Null, Undefined, Symbol

  • Symbol
  1. 对用字符串表示属性名的扩展
  2. 每一个 Symbol 都是唯一的
let s1 = Symbol();
let s2 = Symbol();
s1 === s2; // false

let s = Symbol('abc');
s; // Symbol('abc')

let s = Symbol();
let o = {};
o[s] = function () {
    console.log(233);
}
o[s](); // 233

# 二. 复杂/对象/引用数据类型

Object Function, Array, Object, Set, Map

  • Set
  1. 是对数组的扩展
  2. Set 中每一个 value 都是唯一的
let s1 = new Set(); // Set(0) {}
let s2 = new Set([1, 2, 3]); // Set(3) {1, 2, 3}
let s3 = new Set([1, 2, 3, 4]); // Set(3) {1, 2, 3}
s3.size; // 3
s3[0]; // undefined
  • Map
  1. 是对对象的扩展
  2. Map 中键值对的键名可以不再只是字符串格式, 可以是任意数据类型
let m1 = new Map();
let o = {};
m1.set(o, [1, 2, 3]); // Map(1) {{…} => Array(3)}
m1.o; // undefined
m1.get(o); // [1, 2, 3]

内建函数(built-in function) == 原生函数(native function) 可以被当作构造函数来使用, 包括: String, Number, Boolean, Object, Array, Function, Date, Error, JSON, Regexp 等

通过 Object.prototype.toString() 查看 [[Class]]

  1. 对于基本类型: string, number, boolean 进行封装对象后, 查看 [[Class]]
  2. 对于基本类型: undefined, null, 没有对应的原生构造函数, 直接查看 [[Class]]

# 4.2 封装

在 Number, String, Boolean 数据类型需要用到原型对象上的一些属性和方法的时候, 会被响应的原生构造函数封装对象

let a = 'Pikachu';
// 封装
a = Object(a);
// 封装
let temp = new String('Pikachu');
a = temp;
temp = null;

# 4.3 拆封

要得到封装对象中的基本类型值，可以使用 valueOf() 函数

var a = new String( "abc" );
var b = new Number( 42 );
var c = new Boolean( true );
a.valueOf(); // "abc"
b.valueOf(); // 42
c.valueOf(); // true

# 六. 数据类型转换

# 6.1 抽象值操作

# 6.1.1 toString

  • 对于基本类型
  1. 0 转换为 '0'
  2. true 转换为 "true"
  3. null 转换为 "null"
  4. undefined 转换为 "undefined"
  • 对于引用类型
  1. 先调用 toString() 方法
  2. 再调用 valueOf() 方法

# 6.1.2 toNumber

  • 对于基本类型
  1. 字符串的处理基本遵循数字常量的相关规则 / 语法, 处理失败时返回 NaN
  2. true 转换为 1, false 转换为 0
  3. null 转换为 0
  4. undefined 转换为 NaN
  • 对于引用类型
  1. 先调用 valueOf() 方法
  2. 再调用 toString() 方法

# 6.1.3 toBoolean

  1. 假值 fasle : undefined, null, +0/-0, NaN, ''
  2. 真值 true : 其它
// valueOf()
({x:1}).valueOf(); // {x: 1}
[1,2,3].valueOf(); // [1, 2, 3]
function fn() {};
fn.valueOf();      // fn() {}
new Date(2010,01,01).valueOf(); // 1264953600000
/^[0-9]$/.valueOf(); // /^[0-9]$/ 

// toString()
({x:1}).toString(); // "[object Object]"
[1, 2, 3].toString(); // "1,2,3"
function fn() {}
fn.toString(); // "function fn() {}"
new Date(2010,01,01).toString(); // "Mon Feb 01 2010 00:00:00 GMT+0800 (中国标准时间)"
/^[0-9]$/.toString(); // "/^[0-9]$/"

  1. 对于引用类型: 先执行 toPrimitive, 如果操作数有字符串, 则执行字符串拼接; 否则执行数字运算

- * / 运算符

  1. 执行数字运算
  • * => 布尔值

  2. for ( .. ; .. ; .. ) 语句中的条件判断表达式（第二个）
  3. while (..) 和 do..while(..) 循环中的条件判断表达式
  4. ? : 中的条件判断表达式

# 6.4 宽松相等和严格相等

  • 宽松相等(loose equals) == 和严格相等(strict equals) ===

前者允许在相等比较中进行强制类型转换，而后者不允许

  • == 的比较规则
  1. 数据类型相同时: 不发生类型转换
  2. 数据类型不同时: 执行 toNumber
// 特殊的
null == undefined; // true
  • == 的安全应用
  1. 如果两边的值中有 true 或者 false, 千万不要使用 ==
  2. 如果两边的值中有 [], "" 或者 0, 尽量不要使用 ==
  3. 这时最好使用 === 来避免不经意的强制类型转换

# 6.5 抽象关系比较

  1. 比较双方首先执行 ToPrimitive
  2. 若比较双方都是字符串: 按字母顺序进行比较
  3. 若比较双方有非字符串: 执行 toNumber 后进行比较
