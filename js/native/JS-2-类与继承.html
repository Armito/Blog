# 二. 继承

# 2.1 类的实例部分和静态部分

# 2.1.1 实例部分属性和方法

可被子类和实例继承

  • 通用写法: 属性写在 constructor 函数中, 方法写在类中
class F {
    constructor(x) {
        this.attr = x;
    }
    testFunc() {
        console.log('This is testFunc');
    }
}
  • 其他写法: 实例属性也可以直接写在类中
class F {
    constructor() {
        console.log(this.x);
    }
    x = '233';
}

# 2.1.2 静态部分属性和方法

可被子类继承, 不能被实例继承

  • 原写法: 在类的外部挂载
class F {

}
F.prop = '233';
F.testFunc = function () {
    console.log('This is testFunction');
}
  • 新写法: 添加 static 关键字
class F {
    constructor() {
        console.log(this.prop); // undefined
        console.log(F.prop); // '233'
    }
    static prop = '233'
    static testFunc() {
        console.log('This is testFunction');
    }
}

# 2.2 super 关键字

  • 作为函数的 super

super() 作为代表调用父类的构造函数, 只用在子类的构造函数之中，用在其他地方就会报错

class Parent {
    constructor() {
        
    }
}
class S extends Parent {
    constructor() {
        super() // 相当于 Parent.prototype.constructor.call(this)
    }
}
  • 作为对象的 super

super 在实例方法中指向父类的原型对象, 在静态方法中指向父类

class Parent { 
    static myMethod(msg) { 
        console.log ('static', msg); 
    }
    myMethod(msg) { 
        console.log ('instance' , msg);
    }
}
class Child extends Parent { 
    static myMethod(msg) { 
        super.myMethod(msg); // 相当于 Parent
    }
    myMethod(msg) { 
        super.myMethod(msg); // 相当于 Parent.prototype
    }
}

Child.myMethod(1); // static 1
let child = new Child;
child.myMethod(2); // instance 2

使用 super 的时候，必须显式指定是作为函数还是作为对象使用，否则会报错

class A{

} 
class B extends A { 
    constructor() { 
        super(); 
        console.log(super); // 报错
    }
}
