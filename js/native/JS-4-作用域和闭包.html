# 作用域与闭包

# 1. 编译原理

程序中的一段源代码在执行之前会经历三个步骤, 统称为 "编译"

  1. 分词/词法分析(Tokenizing/Lexing): 将字符串分解成词法单元(token)
  2. 解析/语法分析(Parsing): 将词法单元流(数组)转换成抽象语法树(Abstract Syntax Tree, AST)
  3. 代码生成: 将 AST 转换成可执行的代码

# 2. 要素

各要素所起到的作用:

  1. 引擎: 负责程序编译及执行的过程
  2. 编译器: 负责编译工作
  3. 作用域: 负责提供一套所有声明的标识符(变量)的查询规则

# 3. 作用域查询规则

LHS 查询: 赋值操作的目标 RHS 查询: 赋值操作的源头

function foo(a) {
    var b = a;
    return a + b; 
}
var c = foo( 2 ); 
// 1. 找出所有的 LHS 查询 3 处: c = ..;, a = 2（隐式变量分配）, b = .. 
// 2. 找出所有的 RHS 查询 4 处: foo(2.., = a;, a .., .. b

# 4. 作用域嵌套

在当前作用域无法找到变量, 引擎就会往外层作用域中继续查找, 直到找到该变量或者抵达最外层作用域为止

# 5. 词法作用域

即定义在词法阶段时标识符所在的作用域

function foo(a) {
    var b = a * 2;
    function bar(c) {
        console.log(a, b, c);
    }
    bar(b * 3);
}
foo(2); // 2, 4, 12
// 全局作用域: 一个标识符 foo
// foo 创建的作用域: 三个标识符 a, bar, b
// bar 创建的作用域: 一个标识符 c

# 6. 函数作用域

通过声明函数来创建作用域, 隐藏代码, 作用域外部不可访问内部, 而内部可以访问外部 但函数名和函数调用会被暴露在外

# 7. 立即执行函数

立即执行函数 (Immediate Invoked Function Expression)

  1. 立即执行函数是一个函数表达式而非函数声明, 不可被外部调用
  2. 立即执行函数的作用域遵循函数作用域的规律

# 8. 块作用域 (Block)

let, const, try...catch 均可创建块作用域 let 关键字将声明的变量与块作用域绑定, 这种绑定可以是隐式的或者显示的 块作用域中, let 没有变量提升, let 关键字之前的代码称为暂时性死区 (temporal dead zone), 该区域中变量不可用

# 9. 闭包 (closure)

  • 发生闭包的条件
  1. 定义在函数内部的函数在其定义的词法作用域之外被调用;
  2. 被调用的内部函数访问了其定义的词法作用域之中的变量;
  • 闭包的作用与缺点

闭包会携带包含它的函数的作用域, 故可以延伸变量的作用范围 正是由于闭包是的变量驻留内存, 故建议只在绝对必要时再考虑使用闭包
