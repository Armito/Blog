# 异步编程

  • 异步编程要解决的问题
  1. 无法通过返回值拿到异步函数处理的结果
  2. 如何按照希望的顺序来执行异步函数

# 一. callback

  • 问题一的解决

用一个函数来封装异步 API，并用回调函数作为参数来接收处理的结果

  • 问题二的解决

作为参数的回调函数在异步API处理结果完成后再调用

function fn(callback) {
    let a;
    setTimeout(function () {
        a = 1;
        callback(a);
    }, 0)
}
fn(res => console.log(res));
  • 回调函数的问题
  1. 会造成回调函数的嵌套, 即回调地狱, 不利于维护和阅读
  2. 大脑对于事情的计划方式是线性的, 阻塞的, 单线程的, 但是回调表达异步流程的方式是非线性的, 非顺序的, 这使得正确推导这样的代码难度很大
  3. 回调会受到控制反转的影响, 从而带来信任问题, 比如回调被调用的次数是否会超出预期, 回调时间过早或过晚等

# 二. Promise

  • 问题一的解决

用 Promise 构造函数来封装异步 API，并返回这个 Promise 实例对象

  • 问题二的解决

Promise 对象链式编程

let p = new Promise(function (resolve, reject) {
    let a;
    setTimeout(function () {
        a = 1;
        try {
            resolve(a);
        } catch (e) {
            reject(e);
        }
    }, 0)
});
p.then(res => console.log(res)).catch(err => console.log(err));
  • Promise.xxx
方法 作用 返回值 备注
Promise.all([p1, p2...]) 将多个 Promise 实例包装成一个新 Promise 实例 一个 Promise 对象 参数状态均为 Fullfilled 才为 Fullfilled, 否则 Rejected
Promise.race([p1, p2...]) 将多个 Promise 实例包装成一个新 Promise 实例 一个 Promise 对象 获得率先改变状态的 Promise 的状态和返回值
Promise.resolve(obj) 将对象转化为 Promise 对象, 状态为 Resolved 一个 Promise 对象 如果参数是 Promise 对象, 什么也不发生
Promise.reject(obj) 将对象转化为 Promise 对象, 状态为 Rejected 一个 Promise 对象 参数会作为 reject 的理由
  • Promise.prototype.xxx
方法 作用 返回值 备注
then(onResolved, onRejected) 处理异步代码的结果 一个新的 Promise 实例
catch(onRejected) 处理异步代码的错误结果 一个新的 Promise 实例
  • Promise 的局限性
  1. Promise 只能被决议一次
  2. Promise 只能有一个完成值或一个拒绝理由
  3. 一旦创建了一个 Promise 并为其注册了完成和 / 或拒绝处理函数, 没有办法从外部停止它的进程

# 三. Generator

  • 问题一的解决

将异步处理放在 yield 关键字的后面, 并在 Generator 外部使用 next() 方法输出

  • 问题二的解决

可以通过调整 yield 的顺序来控制异步处理的顺序

function* fn(a) {
    let b = yield new Promise(function (resolve, reject) {
        setTimeout(function () {
            a += 1;
            resolve(a)
        }, 2000);
    }) 
    yield setTimeout(function () {
        console.log(b * 2)
    }, 2000)
    return;
}
let it = fn(4);
it.next().value.then(data => {
    it.next(data); // 10
});
  • Generator 的数据交换

yield 语句本身没有返回值, 或者说总是返回 undefined next() 方法可以带有一个参数, 该参数会被当做上一条 yield 语句的返回值

function* fn() {
    for (let i = 0; true; i++) {
        let reset = yield i;
        if (reset) {
            i = -1;
        }
    }
}
let it = fn();
it.next(); // 0
it.next(); // 1
it.next(true); // 0
  • Generator 的错误处理

Generator 函数内可以部署错误处理代码，捕获函数体外抛出的错误. 两者实现了时间和空间上的分离

function* fn(x) {
    try {
        var y = yield x + 2;
    } catch (e) {
        console,log(e)
    }
    console.log(y)
}
let it = fn(1); 
it.next() // 3
it.throw('error') // 'error'
  • Generator 与 Iterator

Generator 函数就是遍历器生成函数, 返回一个 Iterator

// 给对象部署 Iterator
let o = {
    name: 'denden',
    age: 12,
    length: 2
}
function* gen() {
    let propKeys = Object.keys(this);
    for (let i of propKeys) {
        yield [key, val] = [i, this[i]];
    }
}
obj[Symbol.iterator] = gen;
for (let k of o) {
    console.log(`${k[0]}:${k[1]}`) // name:denden age:12 length:2
}

# 四. async

async 函数是 Generator 函数的语法糖

改进体现在:

  1. 内置执行器: Generator 函数的执行必须靠执行器, async 函数自带执行器
  2. 更好的语义: async await 比起星号和 yield ，语义更清楚
  3. 返回值是 Promise: 比 Generator 函数的返回值是 Iterator 更方便, 可以用 then 方法
var asyncReadFile = async function () {
    var fl = await readFile ('/etc/fstab'); 
    var f2 = await readFile ('/etc/shells');
    console.log(fl.toString()); 
    console.log(f2.toString()); 
}
  • await
  1. await 后面跟一个表达式, 它的作用是等待一个返回值, 这个返回值不一定仍是 promise
  2. 这个表达式可以是一个 promise, 也可以不是
  3. 如果是成功的 promise, await 等待的是这个成功 resolved 的 promise 的值
  4. 如果是失败的 promise, 则必须用 try...catch... 捕获这个失败 rejected 的 promise 的值, 否则抛出错误
  5. 可以使用 return 返回 resolved 的值, 使用 throw 返回 rejected 的值
  6. 如果不是 promise, 则 await 等待的就是这个表达式的值
  7. await 会阻塞下方的代码, 并将它们封装在一个 promise 对象中异步执行
  8. 正因为 await 可能会阻塞代码, 所以必须用在 async 函数中异步执行
