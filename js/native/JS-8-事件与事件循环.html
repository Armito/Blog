# 事件循环

# 1. 定义

Event Loop是一个执行模型

# 1.1 执行/调用栈

CallStack 一个执行函数的栈结构, 遵循先进后出的原则

# 1.2 宏任务

Macrotask => tasks => macrotask Queue 包括: setTimeout, setInterval, setImmediate (Node 独有), requestAnimationFrame (浏览器独有), UI rending (浏览器独有), I/O, DOM 事件, Ajax

# 1.3 微任务

Microtask => jobs => microtask Queue 包括: Promise, MutationObserver, process.nextTick (Node 独有)

# 2. 浏览器的 Event Loop

浏览器的 Event Loop 是 HTML5 规范明确定义的

  • 具体步骤
  1. 执行全局 Script 同步代码, 这些同步代码有一些是同步语句, 有一些是异步语句
  2. 此时, CallStack 会清空
  3. 依次从 microtask queue 的队首取出回调任务, 放入 CallStack 中执行, 直到把 microtask queue 中的所有任务都执行完毕
  4. 如果在执行 microtask 的过程中, 又产生了 microtask, 那么会加入到队列的末尾, 也会在同一个周期被调用执行
  5. 此时 CallStack 和 microtask queue 均为空 5.5 (此时, 若有 UI rending 则会先执行)
  6. 取出 macrotask queue 中位于队首的任务, 放入 CallStack 中执行
  7. 此时, CallStack 会清空
  8. 重复 3 ~ 7, 直到 CallStack, microtask queue 和 macrotask queue 均为空

# 3. NodeJS 的 Event Loop

NodeJS 的 Event Loop 是基于 libuv 实现的

  • NodeJS 中的 macrotask queue

按照执行顺序:

  1. Timers Queue
  2. I/O Callbacks Queue
  3. Check Queue
  4. Close Callbacks Queue
  • NodeJS 中的 microtask queue

按照执行顺序:

  1. Next Tick Queue
  2. Other Microtask Queue
  • 具体步骤
  1. 执行全局 Script 的同步代码
  2. 执行 microtask queue, 先执行所有 Next Tick Queue 中的所有任务, 再执行 Other Microtask Queue 中的所有任务
  3. 执行 macrotask queue 第一阶段 timers 阶段, 执行 Timers Queue 全部任务: setTimeout, setInterval 预定的 callback
  4. 执行步骤 2
  5. 执行 macrotask queue 第二阶段 I/O callbacks 阶段, 执行 I/O Callbacks Queue 全部任务: 除 close 事件外的所有 callback
  6. 执行步骤 2
  7. 执行 macrotask queue 第三阶段 idle, prepare 阶段, 仅 node 内部使用
  8. 执行步骤 2
  9. 执行 macrotask queue 第四阶段 poll 阶段, 获取新的 I/O 事件
  10. 执行步骤 2
  11. 执行 macrotask queue 第五阶段 check 阶段, 执行 Check Queue 全部任务: setImmediate 预定的 callback
  12. 执行步骤 2
  13. 执行 macrotask queue 第六阶段 close callbacks 阶段, 执行 Close Callbacks Queue 全部任务: close 事件的 callback
  14. 执行步骤 2
  15. 重复 3 ~ 14, 直到 CallStack, microtask queue 和 macrotask queue 均为空
