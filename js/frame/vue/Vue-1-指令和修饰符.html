# Vue 指令

指令的本质就是标签的自定义属性

# 一. 闪动问题

  • v-cloak

无属性值, 解决插值表达式的'闪动'问题

[v-cloak] {
    display: none
}

<div v-cloak>{{msg}}</div>

# 二. 数据

  • v-text

有属性值, 即数据, 在指定元素中填充 text, 不能解析标签, 且没有'闪动'问题

  • v-html

有属性值, 即数据, 在指定元素中填充 html, 且能解析标签, 且没有'闪动'问题

安全问题: 同源数据可用, 跨域数据不可用

  • v-pre

无属性值, 指令 + 插值表达式, 让标签内容原文输出

  • v-once

无属性值, 指令 + 插值表达式, 让对应标签的插值表达式的数据不再具备响应式效果, 即只编译一次, 提高性能

  • v-model
  1. 对于输入框: 值是表单中填充 text, 能覆盖 value 属性的值, 且双向绑定数据
  2. 对于单选框: 值是 number, 需要配合 value 属性, 决定 checked 属性, 不能代替 name 属性
  3. 对于复选框: 值是 array, 需要配合 value 属性, 决定 checked 属性, 不能代替 name 属性
  4. 对于下拉框: 1. 单选时: 值是 number, 需要配合 option 的 value 属性, 决定 selected 属性
  5. 对于下拉框: 2. 复选时: 值是 array, 需要配合 option 的 value 属性, 决定 selected 属性
  6. 对于富文本: 值是表单中填充 text, 且双向绑定数据
  • MVVM 思想:
  1. M: Model, Plain Javascript Object, 即数据
  2. V: View, 即 DOM
  3. VW: View-Model, 即 Vue
  • 表单修饰符
  1. .number: v-model.number
  2. .trim: v-model.trim
  3. .lazy: v-model.lazy, 将 input 事件转换为 change 事件

# 三. 事件

  • v-on

# 3.1 事件绑定

元素两种绑定方法:

  1. v-on:事件名称 = "事件处理语句"
  2. @事件名称 = "事件处理语句"
  3. 事件处理语句符合 javascript 语法
  4. 事件处理语句中用 $event 指代事件对象

# 3.2 事件处理函数

事件处理函数调用:

  1. 无参: v-on:事件名称 = "事件处理函数名"
  2. 有参: v-on:事件名称 = "事件处理函数名(prams)"
  3. 事件处理函数中的 this 指向 Vue 的实例对象

# 3.3 事件对象

事件对象的表示方法:

  1. 无参: 事件处理函数形参即为事件对象, 接收的形参名可以任意
  2. 有参: 最后一个参数为事件对象, 且必须以 $event 的实参进行传递, 接收的形参名可以任意

# 3.4 事件修饰符

  1. .stop 阻止事件冒泡:
  2. .prevent 阻止默认行为
  3. .once 只触发一次回调
  4. .self 只自身触发时执行回调
  5. .left 只左键点击时执行回调
  6. .right 只右键点击时执行回调
  7. 串联写法: v-on:事件名称.stop.prevent = "事件处理语句/函数"

# 3.5 键盘事件的按键修饰符

  1. .enter 键盘事件绑定 enter 键
  2. .deldte 给键盘事件绑定 delete 键
  3. .space 给键盘事件绑定 space 键
  4. .tab 给键盘事件绑定 tab 键
  5. 自定义: Vue.config.keyCodes.自定义名称 = 指定按键对应的 event.keyCode 的值

# 四. 属性

  • v-bind

# 4.1 绑定一般属性

  1. v-bind:属性名 = "属性值"
  2. 简写: :属性名 = "属性值"

# 4.2 绑定 class

  1. 基本用法: v-bind:class = "'类名"
  2. 对象用法: v-bind:class = "{类名 1: 标识符 1, 类名 2: 标识符 2...}", 在 data 中操作标识符(存储一个布尔值)
  3. 数组用法: v-bind:class = "[类名变量 1, 类名变量 2...]", 在 data 中操作类名变量(存储一个类名)
  1. 对象的简写形式: v-bind:class = "类名对象", 类名对象 = {类名 1: boolean, 类名 2: boolean...}, 在 data 中操作类名
  2. 数组的简写形式: v-bind:class = "类名数组", 类名变量数组 = [类名 1, 类名 2...], 在 data 中操作类名
  1. 对象数组结合: v-bind:class = "[类名变量 1, {类名 1: 标识符 1}...]"
  1. 默认的 class 属性不会受到影响, 层叠性适用

# 4.3 绑定 style

  1. 对象用法: v-bind:style = "{属性 1: 属性值变量 1, 属性 2: 属性值变量 2...}", 在 data 中操作属性值变量
  2. 简化用法: v-bind:style = "样式对象", 样式对象 = {属性 1: 属性值 1, 属性 2: 属性值 2...}, 在 data 中操作属性
  3. 数组用法: v-bind:style = "[样式对象 1, 样式对象 2...]", 层叠性适用

# 4.4 显示隐藏

  • v-show

有属性值, 是一个布尔值, 控制 DOM 的显示与隐藏

# 五. 语句

# 5.1 条件语句

  • v-if, v-else-if, v-else

有属性值, 用来判断, 分支结构渲染 DOM

# 5.2 循环语句

  • v-for

有属性值 遍历数组:

  1. v-for="item in dataArray"
  2. v-for="(item, index) in dataArray"
  3. v-for="(item, index) in dataArray" :key="index", key 可以帮助 Vue 提高性能

遍历对象: v-for="(value, key, index) in dataObject"

# 六. 自定义指令

# 6.1 全局指令

  • 无值
Vue.directive('指令名', {
    // inserted 为钩子函数, 在被绑定元素插入父节点时调用
    inserted: function (el) {
        // el 为添加指令的元素
    }
})
  • 有值
Vue.directive('指令名', {
    // bind: 钩子函数, 只调用一次，指令第一次绑定到元素时调用
    bind: function (el, binding, vnode, oldVnode) {
        // el 为添加指令的元素
        // binding 为被绑定的数据对象, 值为 binding.value
        // vnode 为虚拟节点
        // oldVnode 为上一个虚拟节点
    }
})

# 6.2 局部指令

在 Vue 实例中定义

const vm = new Vue({
    directives: {
        指令名: {
            bind: function (el, [binding], [vnode], [oldVnode]) {

            }
        }
    }
})

# 6.3 简写

在 bind 和 update 时有相同效果, 不考虑其他钩子函数时使用

Vue.directive('指令名', function (el, [binding], [vnode], [oldVnode]) {

})

const vm = new Vue({
    directives: {
        指令名: function (el, [binding], [vnode], [oldVnode]) {
   
        }
    }
})

# 6.4 钩子函数

  1. bind: 只调用一次，指令第一次绑定到元素时调用
  2. inserted: 被绑定元素插入父节点时调用
  3. update: 所在组件的 VNode 更新时调用
  4. componentUpdated: 指令所在组件的 VNode 及其子 VNode 全部更新后调用
  5. unbind: 只调用一次，指令与元素解绑时调用
