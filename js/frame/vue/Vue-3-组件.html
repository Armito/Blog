# 二. 组件语法

  1. 在 Vue 实例中添加子组件: 使用全局定义和局部定义的组件标签在 DOM 中添加
  2. 在组件中添加子组件: 使用全局定义和在自己内部局部定义的组件标签在模板中添加
  3. 在组件中添加非组件: 使用插槽

# 2.1 定义组件

  1. 一个组件可以看作是一个 Vue 实例对象, 它们具有相同的属性
  2. 组件可以作为元素标签直接放入 DOM, 也可以作为字符串放入组件模板中
  3. 组件必须放在 Vue 的实例或者其他组件中才有效

# 2.1.1 全局定义

  1. 全局定义的组件元素标签可以用在 DOM 的任何非组件内部的位置
  2. 可以用在任何其他组件的模板中
Vue.component('组件名', {
  data: function () {
    return {}
  },
  props: [],
  template: '',
})

# 2.1.2 局部定义

  1. 定义在 Vue 实例中: 作为标签用在 DOM 中的实例内部
  2. 定义在组件中: 作为字符串用在父组件的模板中
const vm = new Vue({
  el: '',
  data: {},
  components: {
    组件名: {
      data: function () {
        return {}
      },
      props: [],
      template: '',
    }
  }
})

# 2.2 data 属性

  1. data 属性必须是一个函数, 且有返回值
  2. 每一个组件实例必须有单独的数据管理

# 2.3 template 属性

  1. template 属性接收一个字符串或模板字符串, 且必须有一个根元素
  2. 组件元素相当于这个根元素
  3. 组件的命名可以是 kebab-case 或 PascalCase
  4. 引用组件到 DOM 时: 只能用 kebab-case 引用, PascalCase 要转化为 kebab-case
  5. 引用组件到模板时: PascalCase 和 kebab-case 均可

# 三. 组件传值

# 3.1 接收父组件的数据: props 属性

  1. props 属性是对 data 属性的扩展, 用在组件自身的模板中
  2. props 属性用来接收其他组件传递的数据
  3. 在 DOM 中组件元素标签上用 v-bind 绑定对应的 props 属性, 并接收父组件数据
<div class="app">
  <menu-com :pipe="title"></menu-com>
</div>

Vue.component('menu-com', {
  data: function () {
    return {}
  },
  props: ['pipe'],
  template: '<div v-text="pipe"></div>'
});
const vm = new Vue({
  el: '.app',
  data: {
    title: 'ABC'
  },
})

# 3.2 接收子组件的数据: 自定义事件

  1. 在子组件中绑定事件后, 触发 $emit() 函数抛出自定义事件, 在子组件的元素标签上监听这个自定义事件
  2. $emit() 函数的第一个参数为自定义事件名, 第二个参数为传递的实参
  3. $emit() 函数只能传递一个实参, 多余的实参会被忽视
  4. 在组件的元素标签上监听这个自定义事件的处理中, 可以用 $event 接收传递过来的实参
<div class="app">
  <menu-com :pipe="title" v-on:handle="handleFun"></menu-com>
</div>

Vue.component('menu-com', {
  data: function () {
    return {}
  },
  props: ['pipe'],
  template: `<div v-text="pipe" v-on:click="$emit('handle', [25, 26])"></div>`
});
const vm = new Vue({
  el: '.app',
  data: {
    title: 'ABC'
  },
  methods: {
    handleFun: function (val) {
      console.log(val) // [25, 26]
    }
  },
})
  1. 接收子组件的传值语法糖: .sync, 相当于:
// 父组件
@update:value=""
// 子组件
this.$emit('update:value', value)
  1. 接收子组件的传值语法糖: v-model, 相当于:
// 父组件
:value="value" @input="value=$event.target.value"
// 子组件
this.$emit('input', value)

# 3.3 接收兄弟组件的数据: 事件中心

  1. 生成一个 Vue 实例作为事件中心, 这种方法只适用于同一个父组件的不同子组件之间
  2. 在子组件的 mounted 函数中, 给事件中心对象绑定接收参数的自定义事件函数: hub.$on('自定义事件名', 形参)
  3. 在另一个子组件中绑定事件, 在其处理函数中触发事件中心对象的传参函数: hub.$emit('自定义事件名', 实参)
  4. 解绑事件: hub.$off('自定义事件名')
<div class="app">
  <menu-com></menu-com>
  <list-com></list-com>
</div>

const hub = new Vue();
Vue.component('menu-com', {
  data: function () {
    return {
      sum: 0,
    }
  },
  template: `
    <div v-text="sum"></div>
  `,
  mounted: function () {
    hub.$on('menu-event', val => {
      this.sum += val;
    });
  }
});
Vue.component('list-com', {
  data: function () {
    return {
      approve: 1,
    }
  },
  template: `
    <div @click="trigger">click</div>
  `,
  methods: {
    trigger: function () {
      hub.$emit('menu-event', this.approve);
    }
  }
});
const vm = new Vue({
  el: '.app'
});

# 四. 组件插槽

  1. 在组件模板中预留的位置添加 slot 标签
  2. 给各个 slot 标签添加 name 属性
  3. 在 DOM 中使用 template 标签包含每块要插入的内容
  4. 使用 v-slot 指令给 template 标签绑定模板中 slot 标签的 name

# 4.1 默认插槽

  1. 默认插槽 template 标签绑定的 name 为 default, 也可以不使用 template 标签或不绑定 name
  2. 在 DOM 中的所有未指明插槽 name 的内容, 都会放到未命名的 slot 标签的位置
<div class="app">
  <div>
    <demo-com>
      <div>插入的内容</div>
    </demo-com>
  </div>
</div>

Vue.component('demo-com', {
  data: function () {
    return {}
  },
  template: `
    <div>
      <div>233</div>
      <slot></slot>
    </div>
  `
});
new Vue({el: '.app'})

# 4.2 具名插槽

  1. 在 DOM 中指明插槽 name 的内容, 都会放到对应 name 的 slot 标签的位置
<div class="app">
  <div>
    <demo-com>
      <template v-slot:nav>
        <div>插入到 nav 位置的内容</div>
      </template>
      <template v-slot:elevator>
        <div>插入到 elevator 位置的内容</div>
      </template>
      <div>插入到默认插槽的内容</div>
    </demo-com>
  </div>
</div>

Vue.component('demo-com', {
  data: function () {
    return {}
  },
  template: `
    <div>
      <ul>
        <li>
          <slot name="elevator"></slot>
        </li>
        <li>
          <slot name="default"></slot>
        </li>
        <li>
          <slot name="nav"></slot>
        </li>
      </ul>
    </div>
  `
});
new Vue({el: '.app'})

# 4.3 作用域插槽

  1. 插入组件的元素需要用到组件的数据
  2. 组件模板中的 slot 标签上使用 v-bind 指令绑定一个自定义属性, 这个自定义属性的值就是要传递的数据
  3. 给称为插槽 prop 对象命名, 通过这个对象可以取到自定义属性传递的数据
<div class="app">
  <div>
    <demo-com>
      <template v-slot:list="slotProp">
        <!-- 插入到 list 位置的内容 -->
        <div :key="i" v-for="(item, i) in slotProp.list">{{item}}</div>
      </template>
    </demo-com>
  </div>
</div>

Vue.component('demo-com', {
  data: function () {
    return {
      list_data: [1, 2, 3]
    }
  },
  template: `
    <div>
      <slot name="list" v-bind:list="list_data"></slot>
    </div>
  `
});
new Vue({el: '.app'})
