# 实例特性

# 一. 计算属性 computed

  1. computed 是对 data 的扩展, 本质都是数据, 两者不能重名, 用法相同
  2. data 中的数据需要处理时可以使用 computed, 达到简化 view 的作用
  3. computed 的数据依赖于 data 中的数据, 是响应式的一种数据监听
  4. computed 的数据必须是一个方法的返回值
  5. computed 的数据会进行缓存, 没有使用或改变时不会调用这个方法
new Vue({
  data: {
    firstName: '',
    lastName: ''
  },
  computed: {
    fullName: function () {
      return this.firstName + ' ' + this.lastName;
    }
  }
})

# 二. 侦听属性 watch

  1. watch 用来监听已有数据的变化
  2. 可以把 watch 看作是数据版的 methods
  3. 大部分数据监听使用 computed 可以满足, watch 用来实现执行异步或开销较大的操作
new Vue({
  data: {
    username: '',
    tip: ''
  },
  watch: {
    username: function () {
      setTimeout(() => {
        if (this.username === 'Armito') {
          this.tip = 'successful';
        } else {
          this.tip = 'failed';
        }
      }, 1000);
      this.tip = 'loading...';
    }
  }
})

# 三. 过滤器 filter

文本格式化, 用在插值表达式中或者 v-bind 表达式中, 以管道符号'|'标识

# 3.1 实例定义

new Vue({
  filters: {
    过滤器名: function (val) {
      return xxx;
    }
  }
})

# 3.2 全局定义

Vue.filter('过滤器名', function (val) {
  return xxx;
})

# 四. 生命周期钩子

  1. beforeCreate: 在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用
  2. created: 在实例创建完成后被立即调用
  3. beforeMount: 在挂载开始之前被调用
  4. mounted: 实例被挂载后调用
  5. beforeUpdate: 数据更新时调用
  6. updated: 由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子
  7. beforeDestroy: 实例销毁之前调用
  8. destroyed: 实例销毁后调用
