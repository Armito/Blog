# HMR(hot module replace)热模块替换

  • 作用：文件发生变化时，只更新发生变化的文件，从而加快打包速度

  • 配置：devServer： { hot: true }

  • 对于css文件，只要有style-loader，默认支持HMR

  • 对于js文件，需要在入口文件手动添加监听

if (module.hot) {
  module.hot.accept('要监听的文件路径', function () {
    // 要监听的语句
  })
}
  • 对于html文件，不支持HMR，且开启HMR会导致html文件热更新失效
// 保证热更新，须在配置文件中更改
entry: ['打包入口文件路径', '需要热更新的html文件路径']

# oneOf

  • 作用：让一个文件只会被一个所匹配到loader处理
modules: {
  rules: [
    { test: xxx, use: xxx },
    { oneOf: [{ test: xxx, use: xxx }, { test: xxx, use: xxx }] }
  ]
}

# 缓存

  • 为什么文件要设置缓存？ --> 因为这样会减少请求，减轻服务器压力

  • 为什么文件要设置hash值？ --> 因为缓存不便于文件紧急修复，加上hash值会让文件名变更，重新请求新文件

  • 为什么要采用contenthash？ --> 因为这样会让文件名根据自身的内容变化，减少打包数量(hash, chunkhash会让若干文件hsah值相同，从而集体更新，重新打包)

  • 设置：文件名.[contenthash:10].后缀名
