# 七. 类

# 7.1 定义类

  1. 可以给变量指定为自定义类型
  2. 类中的构造函数为: 类名(){}
  3. 类中的命名构造函数: 类名.命名构造函数名(){}
  4. get 计算属性, 不能有小括号, 但必须有返回值, 是属性不是方法
  5. set 计算属性, 必须有形参, 是属性不是方法
  6. 类中构造函数的初始化列表: 类名():类属性=值{}

# 7.2 类模块

  1. import '路径'
  2. 私有属性: _属性/_方法, 必须在模块中才能不被直接访问, 但可以被公有方法间接访问或调用

# 7.3 静态属性

  1. 关键字: static
  2. 静态方法可以访问静态属性和调用静态方法
  3. 实例方法可以访问静态属性和调用静态方法
  4. 静态方法不能访问实例属性和调用实例方法

# 7.4 属性的链式操作

dart 中可以用 .. 链式操作类的属性

class Person {
  double width;
  double height;
  Person(double w, double h) {
    this.width = w;
    this.height = h;
  }
  get area {
    return this.width * this.height;
  }
  set areaWidth(val) {
    this.width = val;
  }
  printArea() {
    print(this.area);
  }
}
void main() {
  Person square = new Person(10.0, 2.5);
  square..areaWidth = 5.0..printArea(); // 12.5
}

# 7.5 继承

  1. 使用 extends 关键字
  2. 子类构造函数给父构造函数传参: 子类名():super(){}
  3. 子类构造函数给父命名构造函数传参: 子类名():super.命名构造函数(){}
  4. 复写父类方法: 可以添加修饰符 @override

# 7.6 多态

  1. 父类中定义的方法不实现, 不同的子类中去实现这个方法就是多态
  2. 子类的实例可以指向父类的引用, 此时这个实例只能访问夫类才有的属性

# 7.7 抽象类

  1. 使用 abstract 关键子定义抽象类
  2. 抽象类不能被实例化, 但可以被继承, 继承的子类可以被实例化
  3. 抽象类中没有实现的方法叫做抽象方法, 继承它的子类中必须实现抽象方法

# 7.8 接口

  1. 抽象类和普通类都可以作为接口使用, 但建议使用抽象类来定义接口
  2. 在类中使用 implements 关键字实现接口
  3. 接口中的属性类中必须有, 类可以有自己其他属性
  4. 一个类可以实现多个接口, 用逗号连接各个接口名

# 7.9. 混入 mixins

  1. 使用关键字 with 实现混入
  2. 可以理解成将多个类混合后形成一个新类, 混合后的类具有所有类中的属性, 且相同属性会被覆盖
  3. 被混入的类中不能有构造函数, 不能是继承自其它类的子类
  4. 继承 extends 和混入 mixins 可以一起使用
  5. 混入后的新类的类型是它本身, 也是被混入的任何一个类的类型

class Person<T> {

  }
}

p.printInfo('233');

泛型接口

abstract class Cache <T> {
  String getKey(String key) {
    return key;
  }
  void setKey(String key, T value) {
    print('$key: $value');
  }
}
class FileCashe<T> implements Cache<T> {
  String getKey(String key) {
    return key;
  }
  void setKey(String key, T value) {
    print('$key: $value');
  }
}
void main() {
  FileCashe fc = new FileCashe<int>();
  fc.setKey('ID', 19910407); // ID: 19910407
}
